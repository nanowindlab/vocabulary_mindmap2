# 통합 IA 및 차세대 시각화 UX 시나리오 명세서 (V8 Final-HD)

> **기획 의도**: 대규모 데이터(8,092건) 탐색 시 발생하는 인지적 과부하를 원천 차단하고, 한국어 학습자에게 최적화된 직관적 명칭과 단계별 정보 노출(Progressive Disclosure) 시스템을 구축합니다.

---

## 1. 단계별 마인드맵 렌더링 설계 (Step-by-step Rendering)

사용자가 처음부터 수천 개의 단어에 압도되지 않도록 '계층적 전개' 방식을 채택합니다.

### 1.1. 초기 상태 (Initial View)
- **노출 범위**: 3대 축(System) 선택 시, 해당 축의 **루트(Root)**와 그 하위 **중분류(Category)** 노드까지만 렌더링함.
- **시각적 특징**: 단어(Term) 노드는 보이지 않는 상태이며, 중분류 노드 옆에 포함된 단어 수가 작은 배지(Badge)로 표시될 수 있음.

### 1.2. 동적 확장 인터랙션 (Category-to-Term Expansion)
- **Trigger**: 특정 **중분류(Category) 노드** 클릭.
- **Logic**: 
  - 클릭된 카테고리 하위의 모든 **단어(Term)** 노드가 방사형(Radial)으로 부드럽게 확장됨.
  - **배타적 전개(Exclusive Expansion)**: 다른 중분류를 클릭하면 이전의 확장된 단어들은 자동으로 접히며(Collapse), 새로운 카테고리가 캔버스 중앙으로 포커싱됨. (화면 복잡도 유지 목적)
- **충돌 방지**: 확장 시 단어 노드들이 기존의 다른 중분류 노드를 덮지 않도록, 물리 엔진(D3 Force)이 주변 노드들을 밀어내며 여유 공간을 확보함.

## 2. 복합 내비게이션 및 상태 동기화 (Complex Navigation)

[탭], [보기 모드], [상세 패널]이 유기적으로 연동되는 시나리오를 정의합니다.

### 2.1. 뷰 모드 전환 및 데이터 유지 (View Switching)
- **시나리오**: '식당' 카테고리에서 마인드맵을 보다가 '리스트' 보기로 전환함.
- **State**: 사용자가 마인드맵에서 특정 단어를 선택(Selected)한 상태였다면, 리스트 뷰로 전환 시에도 해당 단어가 목록 최상단에 하이라이트된 상태로 유지되어야 함.
- **플래시카드 연동**: 현재 보고 있는 카테고리(예: 쇼핑 > 결제)에서 '플래시카드' 전환 시, 해당 카테고리의 단어들만으로 즉시 학습 세트가 구성됨.

### 2.2. 연관 단어(Cross-link) 이동 시나리오
- **Action**: 상세 패널의 '교차 연결 장면' 클릭.
- **Feedback**:
  1. 타겟 단어가 다른 축(System)일 경우 자동으로 상단 탭 전환.
  2. 타겟 단어가 속한 중분류 노드를 **자동 확장**하고 단어 노드를 중앙에 정렬(Zoom-to-node).
  3. 시각적 강조(Highlight) 효과 2초간 유지.

## 3. 브랜딩 및 데이터 노출 정책 (Branding & Exposure)

외래어 중심 표현을 지양하고 한국어 학습자에게 친숙한 용어를 사용합니다.

### 3.1. 어휘 중요도(Band) 한국어 명칭 제안
기존 'Band 1~5' 대신 다음과 같은 직관적 명칭을 UI에 적용합니다.

| 기존 명칭 | 제안 명칭 (UI Badge) | 학습적 의미 |
| :--- | :--- | :--- |
| Band 1 | **최상위 필수** | 한국어 생활에서 가장 먼저 익혀야 할 단어 |
| Band 2 | **핵심 중요** | 의사소통의 핵심이 되는 주요 단어 |
| Band 3 | **일상 활용** | 일반적인 대화에서 빈번히 쓰이는 단어 |
| Band 4 | **기초 보조** | 문장을 풍부하게 해주는 기본 단어 |
| Band 5 | **심화/전문** | 특정 상황이나 고급 수준의 심화 단어 |

### 3.2. 빈도수 비공개 및 언어 기호화
- **빈도수 숫자(Raw Number)**: "1,234회 출현"과 같은 수치는 UI 어디에서도 노출하지 않음. 오직 위 명칭과 배지 색상으로만 정보 제공.
- **다국어 기호화**: 영어(ENG), 중국어(CHN), 일본어(JPN) 등 다국어 필드 앞에 국가 국기 아이콘이나 ISO 약어 기호를 배치하여 가독성 향상.

## 4. 개발 에이전트 구현 시 우선순위

1.  **[High]** 중분류 클릭 시 단어 노드만 확장되는 '단계별 렌더링' 엔진 수정.
2.  **[Medium]** 사이드바 계층 메뉴를 중분류(Root 하위) 레벨부터 노출하도록 UI 개편.
3.  **[Low]** 상세 패널에서 '통계' 탭을 삭제하고 어휘 중요도 명칭을 한국어로 교체.
