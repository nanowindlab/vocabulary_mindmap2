## 2. 통합 내비게이션 및 횡단 시나리오

### 2.1. 3단계 계층 탐색 (Hierarchical Navigation)
- **Flow**: System (Tab) ➔ Root (Mindmap Center) ➔ Category (3 Depth) ➔ Term (Leaf)
- **Interaction**:
  - 사이드바에서 Category 클릭 시, 마인드맵 캔버스는 해당 카테고리 노드를 중심으로 **자동 Zoom-in** 처리.
  - 마인드맵에서 Term 클릭 시, 우측 상세 패널(`DetailPanel`) 즉시 로드.

### 2.2. 자동 탭 횡단 라우팅 (Cross-link Routing)
- **시나리오**: [상황과 장소]의 '음식' 상세 정보를 보던 중, 연관 어휘로 [마음과 표현]의 '맛있다'를 클릭함.
- **기술 명세**:
  1. `setActiveTab('expression')` 상태 변경.
  2. `MindmapCanvas`는 '맛있다'가 속한 Root('의견과 가치')와 Category('주관적 가치 평가')를 식별하여 트리 전개.
  3. 타겟 단어('맛있다')를 화면 중앙에 배치하고 하이라이트 효과 적용.

## 3. 콘텐츠 노출 및 품질 정책

### 3.1. 6종 예문 노출 우선순위 (Sentence Priority)
상세 패널의 '예문' 탭에서는 다음 순서에 따라 데이터를 정렬하여 노출한다.
1.  **TOPIK 예문**: 시험 대비 학습자에게 가장 신뢰도 높은 예문.
2.  **국어사전 예문**: 표준적인 쓰임을 보여주는 정석 예문.
3.  **구어체/문어체 예문**: 상황(상황과 장소 탭)에 따라 구어체 예문을 우선 노출할 수 있음.
4.  **AI 생성 보조 예문**: 실생활 빈도가 높지만 사전 예문이 부족한 경우 노출.

### 3.2. 'Band' 등급의 학습적 재정의
단순 빈도 통계를 학습 단계와 연동하여 배제(Badge)로 표시한다.
- **Band 1 (Beginner)**: 일상생활 필수 기초 어휘 (예: 먹다, 집, 나)
- **Band 2~3 (Intermediate)**: 사회적 관계 및 일상 묘사 어휘 (예: 사귀다, 복잡하다)
- **Band 4~5 (Advanced)**: 추상적 논리 및 전문 상황 어휘 (예: 성취하다, 인프라)

## 4. 용어 정제 및 브랜딩 가이드

사용자에게 더 직관적이고 세련된 인상을 주기 위해 UI 용어를 다음과 같이 통일한다.

- **[카드 학습] ➔ [플래시카드] (Flashcards)**
  - '학습'이라는 무거운 느낌보다 빠르고 직관적인 암기 도구라는 인상 부여.
- **[장면 코어] ➔ [상황과 장소]**
- **[표현 코어] ➔ [마음과 표현]**
- **[메타 학습] ➔ [구조와 기초]**

## 5. 실행 블루프린트 (개발 에이전트 구현 가이드)

1.  **상태 유지**: 마인드맵 ↔ 리스트 ↔ 플래시카드 뷰 전환 시, `selectedTermId` 상태를 전역적으로 유지하여 사용자가 보던 단어를 놓치지 않게 함.
2.  **데이터 분리**: 8,545건의 부하를 줄이기 위해 `public/data/`의 JSON 파일을 3대 축별로 물리적 분리하여 로드할 것.
3.  **렌더링 최적화**: 3 Depth 카테고리 구조를 활용하여 한 번에 캔버스에 그려지는 노드의 수를 150개 이하로 제한할 것.
