# IA V4 아키텍처 리팩토링 및 3 Depth 구조화 전략 (RESTART-V1)

> Date: `2026-03-11`
> Status: `Approved by Manager`
> Target: 전 에이전트 공통 지침

## 1. 배경 및 문제 상황 (Background)
현재 IA V4 체계(3대 축, 22개 루트)를 바탕으로 데이터 분류 및 확장이 진행되었으나, 프론트엔드 연동 과정에서 다음과 같은 아키텍처 불일치 및 성능 문제가 발견되었습니다.
1. **프론트엔드 과부하 (3 Depth 유실):** 데이터 에이전트의 LLM 분류 시 1단계(System)와 2단계(Root)까지만 추출되고 3단계(Category/개념)가 유실되었습니다. 그 결과 2단계 폴더 아래 수천 개의 단어가 한꺼번에 렌더링되면서 성능 저하 및 가독성 훼손이 발생했습니다.
2. **단일 거대 파일의 비효율성:** 8,000건이 넘는 단어가 하나의 거대한 `APP_READY_CORE_TREE_V1.json` (약 4.1MB)에 담겨 있어 로딩이 느리고, 3대 축(상황과 장소, 마음과 표현, 구조와 기초)을 독립적인 UI 탭으로 분배하는 라우팅 로직이 무겁습니다.

## 2. 해결 전략: "통합 원천 풀 구성 후 3 Depth 강제화 및 3개 파일 분리"

### 단계 1: 마스터 풀(Master Pool) 구축
*   파편화된 모든 중간 산출물(기존 V1-REV-19 결과물 + V1-REV-21에서 식별/분류된 추가 누락 단어 8,545건 및 각 Candidate 큐)을 단일한 마스터 JSONL 풀로 강제 병합합니다. (단, 영구 제외된 EXCLUDED는 제외).
*   **목표:** 중간 산출물 관리의 복잡성을 제거하고 무결성 있는 단일 데이터 소스를 확립합니다.

### 단계 2: 프롬프트 재설계 및 3 Depth 전수 재분류 (데이터 에이전트)
*   **기획/데이터:** IA V4 명세서를 기반으로, 각 2단계 루트(예: '사람과 관계') 하위에 속하는 구체적인 3단계 개념(Category - 예: 가족, 호칭 등) 리스트를 확정하여 프롬프트에 주입합니다.
*   구축된 마스터 풀을 대상으로, **1단계(System) > 2단계(Root) > 3단계(Category)** 의 3 Depth 스키마를 완벽히 갖추도록 100% 전수 재분류(Triage)를 1-Shot으로 실행합니다.

### 단계 3: 3대 축 기준 물리적 파일 분리 (데이터 에이전트)
*   분류가 완료된 최종 산출물을 하나의 거대 파일로 합치지 않고, `system` 필드(3대 축)를 기준으로 다음 3개의 파일로 쪼개어 생성합니다.
    1.  `APP_READY_SITUATIONS_TREE.json` (상황과 장소)
    2.  `APP_READY_EXPRESSIONS_TREE.json` (마음과 표현)
    3.  `APP_READY_BASICS_TREE.json` (구조와 기초)
*   단, 통합 검색을 위한 `APP_READY_SEARCH_INDEX.json`은 하나로 유지합니다.

### 단계 4: 탭별 지연 로딩(Lazy Loading) 및 UI 렌더링 (개발 에이전트)
*   프론트엔드(`App.jsx`)를 개편하여, 단일 파일을 로드한 후 분배하는 로직을 폐기합니다.
*   각 탭(상황과 장소 / 마음과 표현 / 구조와 기초) 진입 시, 매칭되는 개별 JSON 트리 파일만 로드(Lazy Load)하도록 최적화합니다.
*   복구된 3 Depth(Category)를 트리 UI의 중간 폴더로 렌더링하여 가독성과 렌더링 성능을 확보합니다.

## 3. 에이전트별 행동 지침 (Action Items)
*   **데이터 에이전트:** 기존 V1-REV-21 단계의 트리 재생성(Hydration) 작업은 즉각 중단하고, 8,545건의 Payload 생성 단계까지만 완료된 상태로 보고합니다. 이후 곧바로 '마스터 풀 구축 및 3 Depth 전수 재분류' 작업에 돌입합니다.
*   **개발 에이전트:** 데이터 에이전트의 3개 파일 분리 산출물이 나오는 즉시, 탭 분리 라우팅 및 3 Depth 렌더링 최적화 개발에 착수합니다.
*   **리뷰 에이전트:** 3 Depth 분류가 적용된 결과물의 품질 및 3개 파일 분리 체계의 정합성을 검증합니다.
